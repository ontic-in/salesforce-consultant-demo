## Boundaries & Notes

### Common Gray Areas

| Request | Decision | Notes |
|---------|----------|-------|
| "We need skills-based routing" | Out of scope | Transform tier feature |
| "Can we have multiple chat deployments?" | Limited | 1 deployment included, additional in Transform |
| "Complex entitlement processes" | Limited | Up to 3 basic processes, complex ones in Transform |
| "Case swarming" | Out of scope | Transform tier feature |
| "Customer surveys" | Out of scope | Transform tier feature |
| "Multiple service processes" | Limited | 1-2 processes, multiple in Transform |
| "Approval workflows" | Out of scope | Transform tier feature (3 workflows) |

### Upgrade Triggers

Recommend Transform tier discussion when client mentions:
- Skills-based routing requirements
- Case swarming for complex issues
- Multiple service processes (>2)
- Advanced approval workflows
- Customer satisfaction surveys
- Service Contracts with complex SLAs
- Advanced automation (>5 flows)
- Scheduled reports and dashboards
- Field Service needs

### Implementation Notes

- Service Console is the centerpiece - spend time on layout optimization
- Knowledge base requires content strategy discussion (separate from implementation)
- Omni-Channel is queue-based, not skills-based (skills = Transform tier)
- Live Agent = 1 deployment (typically web chat), additional deployments in Transform
- Entitlement processes should focus on time-based milestones, not complex logic
- Macros should automate common agent tasks (close case, send template, update fields)
- Quick Text focuses on common responses, not full knowledge articles
- Flow limit of 5 means prioritization - focus on highest-impact automations

### User Limits & Constraints

- Up to 25 full Service Cloud users
- 5 custom dashboards maximum (vs 3 in Enable)
- 1 Live Agent chat deployment
- Up to 3 entitlement processes
- Up to 5 Flow Builder automations
- Up to 50 Quick Text messages
- Up to 20 macros
- Basic Omni-Channel (queue-based, not skills-based)
