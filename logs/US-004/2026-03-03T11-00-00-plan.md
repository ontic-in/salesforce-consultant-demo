## Requirements Summary

Relay Logic needs three record-triggered flows to automate case handling, plus a Senior Support Queue as a prerequisite for Flow 1:

1. **VIP Detection & Alert** (After Save) — Gold/Platinum customer with 2+ open cases → Critical priority, route to Senior Support Queue, post Chatter alert
2. **New Customer Prioritization** (Before Save) — Account < 30 days old → set priority to High (VIP overrides if both apply)
3. **Auto-Task Creation** (After Save) — Case assigned to agent → create "Initial Response Required" task with priority-based due date
4. **Senior Support Queue** — Members: Marcus Webb, Priya Sharma, Jake Torres; Object: Case

### Step 3: Create and Deploy Flow 2 — New Customer Prioritization (Before Save)
**Action**: Create Record-Triggered Flow (Before Save) on Case. Runs before Flow 1 (natural execution order). Sets Priority = High for accounts < 30 days old.
**File**: `force-app/main/default/flows/New_Customer_Priority.flow-meta.xml`
**Design**:
- Trigger: Before Insert on Case
- Entry Condition: Account.Account_Start_Date__c != null
- Formula: Calculate TODAY() - 30 days
- Decision: If Account.Account_Start_Date__c > (TODAY - 30)
- Assignment: Set Priority = High (direct $Record update, no DML)
- Status: Active
**CLI Command**:
```bash
sf project deploy start --metadata "Flow:New_Customer_Priority" --target-org deborah@realfast.ai.deb-testing
```
**Verification**:
```bash
sf data query --query "SELECT ApiName, ActiveVersionId, Label FROM FlowDefinitionView WHERE ApiName = 'New_Customer_Priority'" --target-org deborah@realfast.ai.deb-testing
```
**Expected Result**: 1 row, ActiveVersionId != null

### Step 4: Create and Deploy Flow 1 — VIP Detection & Alert (After Save)

**File**: `force-app/main/default/flows/VIP_Detection_Alert.flow-meta.xml`

### Step 5: Create and Deploy Flow 3 — Auto-Task Creation (After Save)

## Known Gotchas

- **Flow status MUST be Active in XML** — `<status>Active</status>` required. Draft flows deploy but don't fire.
- **Use `--metadata` for single-flow deploys** — `--source-dir` deploys ALL flows in directory; broken flows fail entire deploy.
- **CLI may report false MetadataTransferError** — Verify with FlowDefinitionView query, not CLI exit code.
- **Before-Save flows use assignments, not recordUpdates** — $Record fields are set directly via assignment elements.
- **Cross-object field references in entry conditions** — Use dot notation: `{!$Record.Account.Customer_Tier__c}`
- **Queue ID in flow** — Flow 1 needs the actual Queue Group ID to set OwnerId. Must query after queue deployment.
- **SOQL propagation lag** — Per US-001 learning, SOQL may show stale values ~30 seconds after deploy.
