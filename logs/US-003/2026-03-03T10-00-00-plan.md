# Implementation Plan: US-003 — Case Object Configuration

## Story Reference
- **Story ID**: US-003
- **Title**: Case Object Configuration
- **Status**: Planning
- **Plan Date**: 2026-03-03T10-00-00

## Requirements Summary

Relay Logic needs the Case object configured with product tagging (Product_Line__c picklist), auto-populated Customer Tier from Account (Customer_Tier__c formula), custom case statuses (adding In Progress, Waiting on Customer, Resolved), and custom case types (Technical Issue / Bug Report, Integration / API Issue, Billing Question, Feature Request, General Inquiry). Five list views are required for agent workflows.

### Acceptance Criteria
- [ ] Product_Line__c picklist field created (values: Core Platform, Integrations, Analytics Module)
- [ ] Customer_Tier__c formula field created (TEXT(Account.Customer_Tier__c))
- [ ] CaseStatus updated with In Progress, Waiting on Customer, Resolved
- [ ] CaseType updated with 5 Relay Logic-specific types
- [ ] Case_Custom_Fields permission set deployed with correct FLS
- [ ] Permission set assigned to all users
- [ ] Page layout updated with new fields (Product_Line__c required, Customer_Tier__c read-only)
- [ ] 5 list views created and shared to all internal users

## Known Gotchas

- **SVS deploy is full replacement** — Values omitted from XML are deactivated. Must include all existing + new values.
- **Formula fields use return type as `<type>`, not `<type>Formula</type>`** — Use `<type>Text</type>` with a `<formula>` element.
- **Required fields + FLS conflict** — Do NOT set `<required>true</required>` on Product_Line__c. Use page layout Required behavior instead.
- **Formula field FLS must be editable=false** — Salesforce silently corrects this but creates source mismatch.
- **FLS is auto-zero on new fields** — Permission set deployment is required even for System Administrator.
- **Deploy specific PS file, not directory** — Deploying entire permissionsets/ dir could fail if other PS files are broken.
- **SOQL propagation lag** — Per US-001 learning, SOQL may show stale values for ~30 seconds after deploy.
