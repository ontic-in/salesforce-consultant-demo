### Step 4: Deploy OWD Changes (Case first, then Account)

**What:** Change Case OWD from Public Read/Write/Transfer → Private, and Account OWD from Public Read/Write → Public Read Only. Contact stays as Controlled By Parent.

**⚠️ Deploy Order:** Case FIRST, then Account. If Account is changed to Read first while Case is still at ReadEditTransfer, Case would be more permissive than Account — violating the dependency constraint.

**Step 4a — Retrieve current object metadata:**
```bash
sf project retrieve start --metadata "CustomObject:Case" --target-org deborah@realfast.ai.deb-testing
sf project retrieve start --metadata "CustomObject:Account" --target-org deborah@realfast.ai.deb-testing
```

**Step 4b — Edit Case sharing model:**
In `force-app/main/default/objects/Case/Case.object-meta.xml`, change:
```xml
<sharingModel>ReadWrite</sharingModel>
```
to:
```xml
<sharingModel>Private</sharingModel>
```
Also set `<externalSharingModel>Private</externalSharingModel>` if present.

**Step 4c — Deploy Case OWD:**
```bash
sf project deploy start --source-dir force-app/main/default/objects/Case --target-org deborah@realfast.ai.deb-testing
```

**Step 4d — Wait 30 seconds** for sharing recalculation to complete.

**Step 4e — Edit Account sharing model:**
In `force-app/main/default/objects/Account/Account.object-meta.xml`, change:
```xml
<sharingModel>ReadWrite</sharingModel>
```
to:
```xml
<sharingModel>Read</sharingModel>
```
Also set `<externalSharingModel>Read</externalSharingModel>` if present.

**Step 4f — Deploy Account OWD:**
```bash
sf project deploy start --source-dir force-app/main/default/objects/Account --target-org deborah@realfast.ai.deb-testing
```

**Verification:**

- **If DefaultCaseAccess is still ReadEditTransfer:** Deploy may have failed — check `sf project deploy report`
- **If deploy error "sharing calculation interferes":** Wait 60 seconds and retry

## Known Gotchas

| # | Gotcha | Source | Mitigation |
|---|--------|--------|------------|
| 1 | Metadata API vs Data API for roles: Data API `sf data create record` does NOT enforce DeveloperName uniqueness — running twice creates duplicates | Role Hierarchy Guide | Use Metadata API deploy for roles (handles updates in place) |
| 2 | Standard "Service Cloud" profile has locked ModifyAllData | Profile Management Guide | Clone from Standard User instead; remove admin permissions from clones |
| 3 | Profile retrieve without CustomField = empty FLS | Profile Management Guide | Include CustomField members if FLS audit needed (not required for this story) |
| 4 | `--manifest` deploys from `force-app/`, ignoring edits in temp dirs | Profile Management Guide | Use `--source-dir` with the actual file path |
| 5 | OWD is on object metadata (`sharingModel`), not in Settings | OWD Guide | Retrieve via `CustomObject`, not `Settings` |
| 6 | SOQL values differ from XML values (e.g., `None` in SOQL = `Private` in XML) | OWD Guide | Reference the SOQL↔XML mapping table |
| 7 | Case OWD cannot exceed Account OWD | OWD Guide | Deploy Case first (to Private), then Account (to Read) |
| 8 | Sharing recalculation blocks concurrent OWD deploys | OWD Guide | Wait 30 seconds between Case and Account OWD deploys |
| 9 | Profile API name ≠ display name ("Admin" = "System Administrator") | Profile Management Guide | Standard User API name is `Standard` |
| 10 | Every user should have a role; users without roles create unpredictable sharing | Role Hierarchy Guide | All 7 users get explicit role assignments |
